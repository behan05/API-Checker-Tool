User Management API Documentation

Overview

The User Management API allows developers to interact with user data using standard HTTP methods. This documentation provides details on how to use each method, including required parameters and example requests.

HTTP Methods

GET

Use the GET method to retrieve user information.

Endpoint

GET /users

Parameters

  • id (optional): The ID of the user to retrieve.

Example Request

GET http://localhost:5000/api/data/users

Example Response

{"users": [
            {
                "id": 1,
                "name": "John Doe",
                "email": "john@example.com"
            },
            {
                "id": 2,
                "name": "Jane Smith",
                "email": "jane@example.com"
            }
          ]
        }

POST

Use the POST method to create a new user.

Endpoint

POST /users

Request Body

  {
      "name": "New User",
      "email": "newuser@example.com"
  }

Example Request

  POST https://api.example.com/users
  Content-Type: application/json
  
  {
      "name": "New User",
      "email": "newuser@example.com"
  }

Example Response

  {
      "id": 3,
      "name": "New User",
      "email": "newuser@example.com"
  }

PATCH

Use the PATCH method to update an existing user.

Endpoint

PATCH /users/{id}

Request Body

  {
      "name": "Updated User"
  }

Example Request

  PATCH https://api.example.com/users/1
  Content-Type: application/json
  
  {
      "name": "Updated User"
  }

Example Response

  {
      "id": 1,
      "name": "Updated User",
      "email": "john@example.com"
  }

Weather API Example

To get weather information, use the following endpoint:

GET Weather

Endpoint

GET /weather

Parameters

  • city: The name of the city for which to retrieve weather information.
  • unit (optional): The unit of measurement (e.g., metric, imperial).

Example Request

  GET https://api.example.com/weather?city=London&unit=metric

Example Response

  {
      "city": "London",
      "temperature": 15,
      "unit": "Celsius",
      "description": "Clear sky"
  }
